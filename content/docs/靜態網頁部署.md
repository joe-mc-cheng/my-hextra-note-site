# Hextra GitHub 部署指南 

這份指南將帶領你從零開始，在 Windows 環境下建立並部署一個專業的 **Hextra** 知識庫網站。我們將流程拆解為：**環境準備**、**專案初始化**、**GitHub 雲端設定**、以及**日常維護**四個階段。

---

# 🚀 Hextra 知識庫從零到一部署全攻略

## 第一階段：環境工具安裝
在 Windows 上，你需要準備好以下三件核心工具：

1. **Hugo Extended (核心引擎)**
   * 下載 `hugo_extended_..._windows-amd64.zip`。
   * 解壓縮至 `C:\Hugo\bin`。
   * **關鍵設定**：將 `C:\Hugo\bin` 加入 Windows 的「系統環境變數 (Path)」中，確保在終端機輸入 `hugo version` 能看到版本號。
2. **Git for Windows (版本控制)**
   * 安裝後讓你能使用 `git` 指令將檔案推送到 GitHub。
3. **GitHub CLI (權限授權)**
   * 建議安裝，透過指令 `gh auth refresh -s workflow` 解決推送 `.github/workflows` 檔案時的權限錯誤。

---

## 第二階段：GitHub 雲端儲存庫設定
在寫代碼前，先去 GitHub 準備好你的「雲端空間」：

1. **建立 Repository**：
   * 在 GitHub 點擊 `New repository`。
   * 命名為 `my-hextra-note-site` (或其他你喜歡的名字)。
   * 設為 **Public**，且**不要**勾選 Initialize README。
2. **開啟 Pages 來源 (最重要)**：
   * 進入 `Settings` > `Pages`。
   * 將 `Build and deployment > Source` 切換為 **GitHub Actions**。
3. **開啟 Workflow 寫入權限**：
   * 進入 `Settings` > `Actions` > `General`。
   * 將 `Workflow permissions` 改為 **Read and write permissions** 並儲存。

---

## 第三階段：本地專案初始化與結構建立
開啟 PowerShell，切換到你想放專案的目錄（例如 `D:\Project`），開始建立專案：

### 1. 建立 Hugo 骨架
```powershell
hugo new site my-hextra-site
cd my-hextra-site
git init
```

### 2. 引入 Hextra 主題 (使用 Submodule)
```powershell
git submodule add https://github.com/imfing/hextra.git themes/hextra
```

### 3. 配置 `hugo.toml` 設定檔
打開根目錄的 `hugo.toml`，內容務必包含以下關鍵路徑：
```toml
baseURL = "https://<你的帳號>.github.io/my-hextra-note-site/"
title = "我的知識庫"
theme = "hextra"

[outputs]
home = ["HTML", "RSS", "JSON"] # 讓搜尋功能生效
```

### 4. 建立內容目錄結構
Hextra 的導覽列是根據 `content` 目錄結構自動生成的。建議結構如下：
* `content/_index.md` (網站首頁)
* `content/docs/` (放置所有筆記的資料夾)
* `content/docs/_index.md` (筆記目錄的介紹頁)

---

## 第四階段：建立 GitHub Actions 自動化腳本
在專案根目錄建立路徑 `.github/workflows/deploy.yml`，貼入以下內容：

```yaml
name: Deploy Hugo site to Pages

on:
  push:
    branches: ["main"]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'
          extended: true

      - name: Build
        run: hugo --minify

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## 第五階段：新增 Markdown 筆記與發布
這是你未來的日常操作流程。

### 1. 新增一份筆記
在 `content/docs/` 下建立一個檔案 `first-note.md`：
```markdown
---
title: "我的第一份筆記"
date: 2026-04-15
---
# 歡迎來到 Hextra
這是我使用 Markdown 撰寫的內容。
```

### 2. 預覽與推送
```powershell
# 本地預覽 (網址為 http://localhost:1313)
hugo server -D

# 推送到 GitHub (三部曲)
git add .
git commit -m "Add my first note"
git push -u origin main
```

---

## 💡 故障排除心法
* **404 錯誤**：檢查 GitHub `Settings > Pages` 是否已選為 `GitHub Actions`。
* **樣式跑掉**：檢查 `hugo.toml` 裡的 `baseURL` 是否包含完整的 `https://` 以及結尾的 `/`。
* **推送失敗**：如果是 `.github/workflows` 權限問題，請執行 `gh auth refresh -s workflow`。

### 如何將文件部署為網站首頁？
你可以將以上內容存為 `content/_index.md`。這樣當別人（或你自己）進入網站網址時，第一眼看到的就是這份完整的操作手冊，既專業又具備參考價值！
